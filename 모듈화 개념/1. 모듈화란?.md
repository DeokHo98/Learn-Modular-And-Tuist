## 모듈?
```swift
모듈을 재사용 가능한 코드의 단위로 일반적으로 하나의 프레임워크나 라이브러리, 또는 앱의 기능을 묶어놓은것을 의미한다.    
Swift에서는 import 키워드로 불러올수 있는 독립적인 코드 집합을 말한다.
Objectve-C의 헤더 파일과 구현파일을 결합한 것과 유사하지만 Swift에서는 인터페이스와 구현이 같은 파일에 포함될 수 있다.

모듈을 나누는것에 대한 기준은 딱히 정해진것은 없다.
기능단위, Feature단위, 화면단위, 탭바단위, Layer단위 등등 회사, 팀에 맞춰서 적당한 모듈을 나누는 설계를 하면 된다.
예를들면
Core, Feature, Common, Libaray, Resouce 등등으로 말이다.
```

## 모듈화?
```swift
Modular Architecture라고도 하는 모듈화는 
하나의 거대한 구조의 어플리케이션을 만드는 대신 여러개의 작은 단위의 모듈을 만들어 이것을 합쳐 하나의 어플리케이션을 만드는것을 말한다.

이러한 모듈화를 통해서 얻을 수 있는 주요 장점은 다음과 같다.

- 코드 재사용성
모듈을 사용하면 특정기능을 여러 프로젝트에서 재사용할 수 있따. 비단 다른프로젝트 뿐만아니라 서로 다른 모듈에서도 이를 사용할 수 있다.
이러한 재사용성은 코드 중복을 줄이고 유지보수를 쉽게 만들어준다.

- 코드 격리
모듈화된 코드는 다른 모듈과는 격리되어 있어 한 모듈의 변경이 다른 모듈에 영향을 주지 않도록 한다.
이는 코드의 안전성을 높이고, 디버깅또한 쉽게 해준다.

- 빌드 시간 단축
프로젝트를 여러 모듈로 나누면 변경된 모듈만 다시 빌드하면 되기 때문에 전체 빌드 시간을 줄일 수 있다.
이러는 개발 생산성을 높이는데 크게 기여한다.
또한 이러한 이득은 SwiftUI에서 빛을 바랜다. SwiftUI의 프리뷰는 편리하지만 아주 큰 단점이 있다. 프로젝트가 클 수록 프리뷰가 동작하지 않을 확률이 높다는것이다.
하지만 모듈화를 통해 빌드범위를 작게 나누게되면 활용성이 높아진다.

- 데모앱
앱의 각 기능을 테스트하기위헤서는 전체앱을 빌드하고 해당 기능까지 여러뎁스를 거친뒤에 테스트해야했지만.
각 기능별로 모듈을 나누게되면 이러한 데모앱을 통해 테스트를 보다 빠르게 할 수 있다.

결론적으로 모듈화를 통해 코드 구조를 개선하고 유지보수를 쉽게하며 재사용성을 높이고 빌드시간을 줄여 생산성을 높히는데 중요한 역할을 하게 된다.
```
